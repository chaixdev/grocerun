# Development

This folder contains durable developer workflow and operational guidance.

## What Belongs Here

- repeatable developer workflow
- contribution and review procedures
- operational philosophy that should guide future work
- guides that remain accurate independent of a single implementation ticket

## What Does Not Belong Here

- test coverage gap analyses
- fixture design drafts
- implementation summaries
- deprecated setup guides
- speculative plans or one-off investigations

Those artifacts belong in `planning/` while active, or `planning/archive/` once
historical.
