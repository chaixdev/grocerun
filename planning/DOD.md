# Definition of Done (DOD)

**Applicable to all tickets unless specified otherwise.**

## Code Quality
- [ ] Code follows project style guidelines (Linting & Formatting pass).
- [ ] No dead code or console logs.
- [ ] Types are strict (No `any` unless absolutely necessary and documented).

## Testing
- [ ] Unit tests written and passing for new business logic.
- [ ] Integration tests written and passing for API/Server Actions.
- [ ] Manual verification performed in a dev environment.

## Functionality
- [ ] Acceptance Criteria defined in the ticket are met.
- [ ] Error states are handled gracefully (UI feedback).
- [ ] Loading states are implemented.

## UI/UX (If applicable)
- [ ] Responsive design verified (Mobile & Desktop).
- [ ] Accessibility basics checked (Keyboard nav, Alt text).
- [ ] Follows Design System (Colors, Typography).

## Documentation
- [ ] Code is self-documenting or commented where complex.
- [ ] `task.md` updated.
